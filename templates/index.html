Como funciona?

O desafio das APIs é uma competição amigável que objetiva o desenvolvimento individual dos envolvidos nas áreas da computação relacionadas às chamadas APIs, sigla do inglês que significa Application Programming Interface e que representa uma maneira de dar acesso a funções programáticas pertencentes a uma determinada aplicação através de um protocolo de comunicação bem estabelecido como HTTP, RTC, STP e outros.

Queremos disponibilizar um ambiente onde os participantes serão motivados, de maneira gameificada à entregarem cada vez mais funcionalidades e que, por sua vez, fomentarão a vontade e a necessidade de um estudo cada vez mais aprofundado sobre APIs.

O desafio funciona começando com o seu registro utilizando o Github, plataforma de hospedagem de repositórios Git que é considerado como um padrão de indústria e age como um portfolio e um currículo para todos que desejam trabalhar no mercado de tecnologia. Através desse registro o seu progresso será contabilizado na plataforma e, ao mesmo tempo, irá fazer com que você deixe o seu código disponível para outros interessados e para seu futuro empregador!

Seguido disso você recebe um conjunto de instruções de como estruturar seu projeto, quais dados você usará e quais objetivos lhe rendem pontos, podendo estes serem obrigatórios, aqueles que devem ser cumpridos para que você possa entregar seu projeto e este ser considerado no ranking, ou adicionais, aqueles que lhe renderam pontos adicionais se cumpridos com sucesso.

Sobre APIs.

Uma API é um conjunto de rotinas estabelecido por um software para permitir o uso de suas funcionalidades por softwares, aplicativos e clientes externos que desejam usar seus serviços sem querer se importar com detalhes de implementação dessas.

No contexto de desenvolvimento web ela diz respeito à funções que agem baseado em mensagens e respostas (expressas em formatos estruturados como XML e JSON) através do protocolo de comunicação HTTP e que, após o advento da web 2.0, tem dado preferência para o padrão REST.

O padrão REST (de Representational State Transfer) é um padrão de comunicação que se baseia em:

  • Ser stateless, isto é, não guardar ou depender de um controle de estados, as mensagens possuem toda a informação necessária para cumprir com a requisição;
  • Implementar um conjunto de operações bem definidos, chamados verbos, sendo estes: GET, POST, DELETE e PUT;
  • Uma identificação única dos recursos, os URIs; e
  • Uso de hipermídia, permitindo a navegação entre os recursos sem mais nenhum outro artifício além de chamadas REST

Atraés da combinação destes conceitos, você irá abordar os desafios propostos para atingir os objetivos desejados e, assim, ater um conhecimento cada vez mais aprofundado sobre este conceito importantíssimo e de uma utilidade inimaginável!
